## 제1장: 존재 이유 — "이것은 만들어야 하는가?"

### 🔍 [질문 1] 사용자의 진짜 문제는 "정보 부족"인가, "행동 부족"인가?

#### ⚡ 아인슈타인 분석

PRD는 사용자의 문제를 **"뉴스 노이즈에 휘둘리는 감정적 매매"**로 정의한다. 이것은 정확한가?

개인 투자자의 실제 행동 패턴을 관찰하면:

| 행동 | 빈도 | 실제 원인 |
|------|------|-----------|
| 폭락 시 패닉 매도 | 높음 | 정보 부족이 아니라 **공포 감정** |
| 급등 시 추격 매수 | 높음 | 분석 부재가 아니라 **FOMO 감정** |
| 좋은 종목을 사놓고 조기 매도 | 매우 높음 | 확신 부족 → **행동의 문제** |
| 분석은 하지만 실행 안 함 | 높음 | 분석 마비(Analysis Paralysis) |

**진단**: 개인 투자자의 문제는 **90%가 행동(심리)의 문제**이고, 10%만 정보의 문제다. 그런데 이 제품은 100% 정보를 제공하는 도구다. 이것은 **환자에게 체온계를 더 좋은 것으로 바꿔주면서 병이 나을 것이라고 말하는 것**과 같다.

#### 🏛️ 소크라테스 반문

> **"정보를 더 주면 더 나은 결정을 한다고 확신하는가?"**

- Sheena Iyengar의 잼 실험: 선택지 24개 → 구매율 3%. 선택지 6개 → 구매율 30%. **정보가 많을수록 행동은 줄어든다.**
- 피델리티의 가장 수익률 좋은 계좌: **계좌 주인이 사망했거나 잊어버린 경우.** 분석과 모니터링이 수익률을 해쳤다.
- 3종 판독기 × 4~5개 지표 × 수십 종목 = **수백 개 데이터 포인트**. 차트 판독기가 "매수"이고 역발상 판독기가 "매도"이면, 사용자는 무엇을 해야 하는가?

#### 💡 개선 제안

```
현재: 3종 판독기 점수 + 세부 지표 + 차트 + 테이블 + 알림 → 사용자가 해석
개선: 한 줄 결론 + 행동 버튼 1개

예시:
  "오늘의 시장: 🟡 중립 (32/60점). 보유 종목 유지. 추가 매수는 30점 이하에서."
  [상세 보기] ← 이것만 눌러야 세부 지표가 보임

핵심: "정보 → 해석 → 결정"의 3단계를 "결론 → (선택적)근거"의 2단계로 축소.
소몽의 철학 "뉴스 대신 지표"를 한 단계 더 밀고 나가면 "지표 대신 결론"이다.
```

### 🔍 [질문 2] Finviz, Koyfin, CNN Fear & Greed가 이미 있는데, 왜 이것이 필요한가?

#### ⚡ 아인슈타인 분석

기존 대안을 정직하게 매핑하면:

| 소몽 판독기 기능 | 무료 대체재 | 소요 시간 |
|-----------------|-----------|----------|
| MA 상회 종목 비중 | barchart.com/market-data | 30초 |
| 이격도·RSI | finviz.com → 종목 차트 | 1분 |
| Forward PE / PEG | yahoo finance → Statistics | 1분 |
| 적정주가 계산 | 엑셀 = EPS × 성장률 | 3분 |
| VIX | google "VIX" | 5초 |
| P/C Ratio | CBOE 웹사이트 | 30초 |
| AAII Sentiment | aaii.com | 30초 |
| **합계** | **3~4개 사이트 조합** | **~7분** |

7분이면 사람이 직접 할 수 있다. **이 7분을 3초로 줄이는 것**이 이 제품의 유일한 존재 이유다.

그런데 현재 MVP의 반응 시간: Yahoo Finance 실시간 호출 → **3~5초 응답**. 캐시 미스 시 더 길어질 수 있다.

**결론**: "7분을 3초로"가 아니라 "7분을 5초로". 가치 제안이 약해진다.

#### 🏛️ 소크라테스 반문

> **"소몽의 방법론이 공개된 강의 콘텐츠라면, 누구든 같은 엑셀을 만들 수 있다. 이 앱의 방어 가능한 해자(moat)는 무엇인가?"**

- 소몽의 네이버 카페와 텔레그램 채널이 **이미 같은 분석을 무료로 제공**한다. 이것이 PRD의 경쟁 분석에서 빠져 있다.
- 소몽 브랜드 없이 독립 서비스로 생존 가능한가? "3종 판독기"는 결국 RSI + 이동평균 + PER + 센티먼트 조합이다. 이것은 어느 투자 서적에도 나온다.
- "통합 점수"가 차별화라면, 그 통합이 개별 지표보다 **더 정확하다는 증거**가 있는가?

#### 💡 개선 제안

```
존재 이유를 "자동화"가 아닌 "행동 촉발"로 재정의:

1. 수동 분석 대체 → 필요하지만 불충분
2. 핵심 가치: "분석 결과를 행동으로 연결하는 파이프라인"
   - 점수가 기준 이하 → 자동 알림 → 매수 체크리스트 제공
   - 점수가 기준 이상 → 자동 알림 → 매도 체크리스트 제공
   - 포트폴리오 연동 → "당신의 AAPL 비중이 30%. 현재 점수 기준 20%가 적정"

3. 해자(Moat) 구축 전략:
   - 사용자 포트폴리오 데이터 축적 → 개인화 → 전환 비용 발생
   - 히스토리컬 점수 트래킹 → 시간이 지날수록 가치 증가
   - 소몽 커뮤니티 연동 → 네트워크 효과
```

## 제2장: 과학적 근거 — "3종 판독기는 작동하는가?"

### 🔍 [질문 3] 3종 판독기의 백테스트 결과는? 시장을 이긴 적이 있는가?

#### ⚡ 아인슈타인 분석 — 현재 구현의 치명적 결함

기존 아인슈타인 검토에서 이미 밝혀진 핵심 결함을 재정리하고 **신규 발견 사항**을 추가한다:

**🔴 결함 1: 차트 판독기가 소몽 원본과 다른 것을 측정하고 있다**

| 지표 | 소몽 원본 (2-2강) | 현재 구현 | 심각도 |
|------|-------------------|----------|--------|
| 핵심 지표 ① | **200일 MA 상회 종목 비중** (barchart.com, 30%/70% 기준) | ❌ **없음** — 개별 종목 MA 위치로 대체 | 🔴 치명적 |
| 핵심 지표 ② | **50일 MA 상회 종목 비중** (동일 소스) | ❌ **없음** | 🔴 치명적 |
| 이격도 기준선 | **200일선** 기준 | 50일선 기준 | 🔴 높음 |
| M/W 패턴 | 소몽 엑셀에 **없음** (보조 참고 수준) | 4번째 핵심 지표로 편입 | 🟡 중간 |

소몽이 **"90% 이상 적중률, 거의 예외 없이"**라고 반복 강조한 지표(MA 상회 종목 비중)가 코드에 **아예 존재하지 않는다**. 이것은 피아노에서 건반을 빼먹은 수준의 설계 오류다.

**🔴 결함 2: 역발상 판독기가 사실상 VIX 단일 지표다**

```
현재 역발상 판독기 5개 지표의 실제 상태:

┌────────────────────┬─────────────────┬──────────────┐
│ 지표               │ 데이터 소스      │ 실제 상태     │
├────────────────────┼─────────────────┼──────────────┤
│ VIX                │ Yahoo Finance   │ ✅ 실제 데이터 │
│ Put/Call Ratio     │ CBOE 스크래핑    │ ⚠️ 실패 빈번, 기본값 0.85 │
│ AAII Bull/Bear     │ 없음            │ ❌ 하드코딩 35/30 │
│ Margin Debt YoY    │ 없음            │ ❌ 하드코딩 0 │
│ HY Spread          │ FRED (API키 필요)│ ⚠️ 키 없으면 기본값 3.5 │
└────────────────────┴─────────────────┴──────────────┘

유효 지표: 1~2개 / 5개
실제 유효 점수: 4~8점 / 20점

→ 이것은 "5종 지표 역발상 판독기"가 아니라 "VIX 한 줄짜리 계산기"
→ 사용자에게 20점 만점의 정교한 분석인 것처럼 보여주는 것은 기만
```

**🔴 결함 3: 적정주가 공식이 절반만 구현됨**

```
소몽 원본 공식:
  적정주가 = 예상 EPS × 적정 PER
  적정 PER = 성장률(%) × 1 ~ 2  (보수적~낙관적 범위)
  강세장:    성장률(%) × 1.5 ~ 2.5

현재 구현:
  적정 PER = 성장률(%) × 1  (하한값만)
  성장률 데이터 0개일 때 → 15% 가정 ← 위험한 디폴트

결과: 모든 적정주가가 보수적 하한값. 실제보다 낮게 산출됨.
      → "거의 모든 종목이 고평가"로 판정되는 시스템 편향 발생
```

#### 🏛️ 소크라테스 반문

> **"백테스트 없이 이 점수 체계가 유효하다고 어떻게 확신하는가?"**

- **과적합(Overfitting) 위험**: 점수 구간(0-15: 극도 공포, 16-25: 공포 등)의 통계적 근거가 없다. 이것이 직관이라면, "소몽 1인의 경험"이지 **일반화 가능한 법칙이 아니다.**
- **생존자 편향**: 소몽의 성공 사례 위에 구축된 시스템. 같은 방법론으로 실패한 사람들의 데이터가 반영되지 않았다.
- **효율적 시장 가설(EMH)과의 충돌**: RSI, 이동평균선, PER 같은 **공개 지표**로 초과수익을 낼 수 있다면, 왜 월가의 퀀트 펀드들이 같은 지표를 쓰면서도 시장을 이기지 못하는가?

> **"점수가 30인 날과 31인 날, 행동이 달라야 하는가? 그 1점의 차이가 의미하는 것은?"**

- RSI 29 → 5점, RSI 31 → 4점. **0.1%의 차이로 20% 점수 변동.** 이 단절 효과(cliff effect)가 잘못된 신호를 만들 수 있다.
- 3개 판독기에 동일 가중치(20점+20점+20점)를 부여하는 근거는? 장세에 따라 밸류에이션이 더 중요할 때도, 센티먼트가 더 중요할 때도 있다.

> **"시장 타이밍이 가능하다는 전제 자체가 맞는가?"**

- **뱅가드 연구(2019)**: 일정 금액 즉시 투자 > 시장 타이밍 노린 투자 (67% 확률)
- **피터 린치**: "시장 타이밍을 맞추려다 잃은 돈이 하락장에서 잃은 돈보다 많다"
- 이 서비스의 핵심 전제가 학술적으로 부정되고 있다면, 존재 이유를 **재정의**해야 한다.

#### 💡 개선 제안

```
1. 즉시 수정 (Critical):
   ─────────────────────
   a) 차트 판독기에 "MA 상회 종목 비중" 데이터 추가
      - 소스: barchart.com 스크래핑 또는 대안 API
      - M/W 패턴 제거 → 이 자리에 MA 상회 비중 투입
   
   b) 이격도 기준을 50일선 → 200일선으로 수정
   
   c) 역발상 판독기의 하드코딩 지표를 UI에서 "미수집" 명시
      - 또는 MVP에서 역발상 판독기를 2지표(VIX + HY Spread)로 축소
      - 20점 만점이 아닌 "수집된 지표 기준 비례 점수" 표시
   
   d) 적정주가를 범위(보수적~낙관적)로 출력
      - fairPrice_low = EPS × growth × 1.0
      - fairPrice_mid = EPS × growth × 1.5
      - fairPrice_high = EPS × growth × 2.0

2. MVP 전 수정 (Important):
   ─────────────────────
   e) 백테스트 실행: 2020 코로나, 2022 인플레 조정 시점에서
      판독기 점수와 실제 시장 움직임의 상관관계 검증
   
   f) 점수 추세 감지 로직 추가
      소몽 원본: "피크를 찍고 내려오기 시작할 때 팔아라"
      → 현재: 스냅샷만 산출 / 개선: 방향 변화 감지 (Δ score)

3. 전제 재정의:
   ─────────────────────
   "이 시스템은 시장 예측 도구가 아니라, 
    장기적으로 랜덤보다 나은 확률적 우위(Edge)를 제공하는 
    의사결정 보조 도구입니다."
   → 이 한 문장이 모든 UI, 마케팅, 면책조항의 기반이 되어야 한다.
```

## 제3장: 기술 설계 — "이 아키텍처가 견딜 수 있는가?"

### 🔍 [질문 4] Yahoo Finance 무료 API에 사업의 운명을 맡겨도 되는가?

#### ⚡ 아인슈타인 분석

```
Yahoo Finance 의존도 분석:

┌─ 차트 데이터 ───── Yahoo Finance (yahoo-finance2) ──── 유일한 소스
├─ 재무 데이터 ───── Yahoo Finance                   ──── 유일한 소스  
├─ VIX ──────────── Yahoo Finance                   ──── 유일한 소스
├─ HY Spread ────── FRED API                        ──── 독립 소스 ✅
└─ 경제 지표 ────── FRED API                        ──── 독립 소스 ✅

결론: 핵심 데이터 3/5가 Yahoo Finance 단일 의존
      Yahoo가 차단되면 서비스의 80%가 중단됨
```

**역사적 선례**:
- 2017년: Yahoo Finance 무료 API 돌연 중단 → 수만 개 개인 프로젝트 사망
- 2024년 11월: Yahoo Finance rate limiting 대폭 강화 (yfinance GitHub Issue #2128)
- `yfinance`는 공식 API가 아니라 **웹 스크래핑 래퍼** → 언제든 차단 가능

코드의 단일 장애점(Single Point of Failure):

```typescript
// yahoo-finance.ts — 이 한 줄이 죽으면 서비스 전체가 죽는다
const yf = await import('yahoo-finance2');
return yf.default;
```

#### 🏛️ 소크라테스 반문

> **"무료 데이터에 의존하면서 유료 서비스를 제공하려는 것의 모순은?"**

**아인슈타인의 면도날**: "Everything should be made as simple as possible, but not simpler."

#### 🏛️ 소크라테스 반문

> **"이 모든 것이 첫 번째 버전에 필요한가? 만약 스프레드시트 한 장으로 제공해야 한다면?"**

```
3. 한 줄 설명이 따라온다.
   예: "AAPL: 🟡 적정가 부근. 급할 필요 없음. (38/60)"
4. 끝.

나머지는 사용자가 이것을 10번 이상 쓴 후에 추가해도 늦지 않다.
```

- 유동성 판단: Phase 2 (시장 전체 신호는 MVP 이후)
- 포트폴리오: Phase 2 (먼저 단일 종목 분석이 가치 증명)
- 종목 검색/TOP 20: Phase 2 (사용자가 직접 종목 입력)
- 알림: Phase 3
- 교육: 별도 채널 (유튜브/블로그)


### 🔍 [질문 6] AI 투자추천의 법적 책임은 누구에게 있는가?

#### ⚡ 아인슈타인 분석

**자본시장법 제7조** 관련 리스크:

```
현재 코드의 법적 위험 표현들:

combined-screener.ts:
  signal: 'strong_buy'      ← 명시적 매수 권유
  signal: 'strong_sell'     ← 명시적 매도 권유
  "공격적 매수 구간입니다"    ← 투자 권유 문구
  "분할 매도를 시작하세요"    ← 행동 지시

PRD 종합 신호 판정표:
  "적극 매도, 현금 80%+"     ← 구체적 비중 지시
  "적극 매수, 주식 80%+"     ← 구체적 비중 지시
```

한국 자본시장법상, **투자자문업 등록 없이** "매수/매도"를 권고하는 서비스를 운영하면 불법이다.

**최악의 시나리오**:  
2008년 같은 폭락장에서 "극도 공포 → 적극 매수" 신호 → 사용자 추가 매수 → 시장 추가 -40% 하락 → 집단소송 → 서비스 중단 + 개인 배상 책임.

#### 🏛️ 소크라테스 반문

> **"면책조항만으로 법적 리스크가 해소되는가?"**

- 미국 SEC도 "robo-advisor"에 투자자문업 등록을 **요구**한다.
- "교육 목적"이라는 표현으로 피하려 해도, `strong_buy` / `strong_sell` 같은 API 응답은 **명시적 투자 권유**로 해석될 수 있다.
- "투자자문"이라는 **서비스 이름 자체**가 법적 문제를 야기할 수 있다.

> **"PEG < 1이 항상 저평가인가? 성장률 예측이 틀리면?"**

- PEG = PE / 성장률. 성장률이 **애널리스트 예측치**에 기반한다.
- 2022년 META: 애널리스트 EPS 컨센서스와 실제의 **50% 이상 괴리**.
- PEG < 1인데 성장률이 과대 추정이었다면, 실제로는 **PEG > 2 (고평가)**.
- 이 오류를 시스템이 경고하지 않으면, 잘못된 "저평가" 신호로 매수한 사용자가 손실.

#### 💡 개선 제안

```
즉시 조치 (법적 리스크 제거):
━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 서비스 명칭 변경:
   "투자자문봇" → "투자 분석 도구" 또는 "투자 교육 보조 도구"

2. API 응답값 변경:
   strong_buy → very_low_valuation  (중립적 서술)
   strong_sell → very_high_valuation
   "적극 매수" → "역사적으로 유사 구간에서 반등 확률 높음 (참고용)"
   
3. 모든 비중 지시 제거:
   "현금 80%+" → "과거 유사 상황에서의 통계 참고"
   
4. 면책 체계:
   - 회원가입 시: 면책 동의서 서명
   - 판독기 결과 페이지: 매번 상단 고지
     "⚠️ 이 정보는 교육 목적의 데이터 분석이며, 투자 권유가 아닙니다."
   - 이용약관: 투자 손실 면책 명시
   
5. 법률 검토:
   - 투자자문업 등록 여부 법률 자문 (1순위)
   - 금융위원회 로보어드바이저 테스트베드 규정 확인

6. 적정주가 신뢰도 표시:
   - 성장률 데이터 0개 → "⚠️ 신뢰도 낮음: 성장률 추정 데이터 부족"
   - 성장률 변동 계수 높은 종목 → "⚠️ 이 종목은 적정주가 산출이 부정확할 수 있습니다"
   - 적자 기업 / IPO 1년 미만 → "❌ 이 종목에는 이 분석 방법이 적합하지 않습니다"
```

#### 🏛️ 소크라테스 반문

- 하지만 `yahoo-finance2`도 동일한 데이터를 제공한다

#### 💡 개선 제안

```
선택지 A (권장): Python 삭제, TypeScript 단일 코드베이스
  장점: 유지보수 비용 절반, 버그 동기화 불필요
  단점: Python의 pandas 편의성 포기
```

### 🔍 [질문 9] 개인투자자가 월가를 이길 수 있다는 전제가 맞는가?

#### ⚡ 아인슈타인 분석

이것은 기술이 아닌 **철학적 전제**의 문제이지만, 제품 설계에 직접적 영향을 미친다.

```
학술 연구 결과:

버튼 말키엘(『랜덤워크 투자 수업』):
  → "원숭이가 눈감고 던진 다트 > 전문가의 종목 선별"

SPIVA 리포트 (S&P Dow Jones, 2024):
  → 액티브 대형주 펀드의 93%가 15년 동안 S&P 500에 미달

워렌 버핏 10년 내기 (2008-2017):
  → S&P 500 인덱스 펀드 vs 헤지펀드 5개
  → S&P 500: +125.8% / 헤지펀드 평균: +36%
  → 프로도 시장을 못 이기는데, 소몽 판독기로?
```

그러나 소몽의 접근법은 "시장을 이기는 것"이 아니라 **"크게 잃지 않는 것"**에 초점이 있다:

```
소몽의 실제 목표 (강의 분석):
  ≠ "S&P 500 수익률을 초과한다"
  = "공포 구간에서 매도하지 않고, 과열 구간에서 추격 매수하지 않는다"
  = "감정적 매매의 손실을 줄인다"
  = 행동 교정 도구
```

#### 🏛️ 소크라테스 반문

> **"그렇다면 이 도구의 성공 지표는 '수익률'이 아니라 '감정적 매매 빈도 감소'여야 하지 않는가?"**

현재 PRD의 KPI:
- "판독기 점수 정확도 (백테스트) > 65% 방향성 적중" → 예측 정확도 (시장 타이밍)
- "적정주가 오차범위 ±15% 이내" → 밸류에이션 정확도

이것은 **"시장을 이기겠다"는 전제**의 KPI다.

> **"과도한 분석이 오히려 행동 마비를 유발하지 않는가?"**

- 판독기 점수가 상충할 때 (차트 = 매수, 역발상 = 매도), 사용자는 아무것도 하지 않게 된다
- "아무것도 하지 않는 것"이 최선인 경우도 많지만, **시스템이 "관망"을 지시하는 것**과 **사용자가 혼란으로 동결되는 것**은 다르다

#### 💡 개선 제안

```
1. KPI 재정의:
   "방향성 적중률 65%" (삭제)
   → "감정적 매매 방지율" (신규)
     - "판독기 점수가 극단(0-15 또는 46-60)일 때 반대 행동을 한 사용자의 비율"
   → "포트폴리오 변동성 감소율" (신규)
     - "판독기 사용 전/후 포트폴리오 max drawdown 비교"

2. 상충 신호 처리:
   차트="매수" + 역발상="매도" → 
   현재: 점수를 합산해서 "중립"
   개선: "⚠️ 신호 상충: 기술적으로는 매수 구간이나 시장 심리가 과열.
          보수적 접근 권장. 소량 분할 매수 또는 관망."
   → 상충 자체를 정보로 전달 (합산하여 의미를 소실시키지 않음)

3. 제품 포지셔닝 변경:
   "투자자문봇" → "투자 행동 교정 도구"
   "이 도구는 당신이 더 좋은 종목을 찾아주지 않습니다.
    당신이 이미 찾은 종목을 잘못된 타이밍에 사고파는 것을 막아줍니다."
```

```typescript
  above50: number;  // S&P 500 중 50일선 위 종목 비율 (%)

  // 1. barchart.com 스크래핑 (소몽 원본 소스)

// 수정된 점수 체계 (소몽 원본)
function calculateMABreadthScore(above200: number, above50: number): number {
  // 200일선 상회 비율
  let score200 = 0;
  if (above200 < 30) score200 = 1;      // 극도 공포 → 역발상 매수
  else if (above200 < 50) score200 = 2;
  else if (above200 < 70) score200 = 3;  // 중립
  else if (above200 < 85) score200 = 4;
  else score200 = 5;                     // 과열 → 주의

  // 50일선도 동일 로직
  // 최종 = (score200 + score50) / 2 → 소몽의 평균 방식
}
```

### C. 점수 추세 감지 — 신규 구현 제안

```typescript
// 소몽: "피크를 찍고 내려오기 시작할 때 팔아라"
function detectScoreTrend(
  history: { date: string; score: number }[]
): 'rising' | 'peak_reversal' | 'falling' | 'bottom_reversal' | 'stable' {
  if (history.length < 3) return 'stable';
  
  const recent3 = history.slice(-3);
  const [a, b, c] = recent3.map(h => h.score);
  
  if (a < b && b > c) return 'peak_reversal';     // △ → 꼭지점에서 하락 시작
  if (a > b && b < c) return 'bottom_reversal';    // ▽ → 바닥에서 반등 시작
  if (a < b && b < c) return 'rising';
  if (a > b && b > c) return 'falling';
  return 'stable';
}

// 사용: "점수가 52에서 48로 하락 전환 → 과열 피크 감지 → 분할 매도 알림"
```
